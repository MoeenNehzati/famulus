# Research Notes: General

- Primary goal:
  - preserve or communicate technical understanding efficiently
- Shared flow expectations:
  - the note should quickly reveal what kind of note it is, why it exists, and what background it assumes
  - lighter motivation is acceptable, but purpose should still be explicit
  - the note should make clear whether it is exploratory, explanatory, status-reporting, or feedback-seeking
  - the note should state what the reader is supposed to do with it: read for context, verify a step, react to a conjecture, or integrate it into a larger draft
- Shared burden:
  - lower motivation than public-facing documents
  - moderate signposting
  - variable rigor/detail
- Shared constraints:
  - notes often rely on context, but should remain interpretable later
  - the reader should be able to tell what they are supposed to do with the note
  - a note should preserve just enough background so that it still makes sense after the shared project context has faded
- Common failure modes:
  - too much reliance on unstated context
  - unclear purpose
  - no indication of how the note relates to the larger project
  - finished claims, tentative ideas, and requests for feedback are not separated clearly
