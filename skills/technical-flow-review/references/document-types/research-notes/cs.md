# Research Notes: Computer Science Deviations

- Relative to the general research-notes baseline:
  - compressed technical content is acceptable if the note's role is clear
  - the note should say early whether it is recording a design idea, algorithm sketch, proof outline, implementation observation, or experimental finding
  - if the note is meant for collaborators, it should isolate the open decision or the main technical obstacle rather than burying it inside background
- Common additional failure mode:
  - no stable indication of what the reader is supposed to take from the note
  - the key idea is present, but never distilled into a statement a collaborator could react to quickly
