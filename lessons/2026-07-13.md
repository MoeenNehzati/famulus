# 2026-07-13

## Blueprint-backed user behavior sources need a tracked graph node even when their content is personal

**Context:** Designing user-level preferences for `skills/email-triage` without committing one user's personal triage rules as canonical repository behavior.

**Observation:** A blueprint behavior-source binding must point to an existing regular file, so an ignored-only preference file cannot provide a stable node on a fresh clone. The smallest local workaround is to commit an empty preference file and its blueprint sidecar, then mark the tracked preference file locally with `git update-index --skip-worktree <path>`. An uppercase `S` from `git ls-files -v -- <path>` confirms the flag; `git update-index --no-skip-worktree <path>` removes it. `assume-unchanged` is not the appropriate mechanism.

**Impact:** This keeps blueprint construction valid and normally hides one user's local preference edits from status and ordinary commits, but it is not a general user-behavior-source abstraction. `skip-worktree` affects only Git's working-tree treatment: blueprint health records include the bound-file hash, so personalizing the file makes the existing audit stale until the local customized state is certified again. The flag is local to one index, is not cloned, is not a privacy or backup boundary, and may be cleared or disrupted when Git must update the tracked file. Checkout, reset, merge, or an upstream change can therefore endanger the personal content.

**Use/Avoid:** Use `skip-worktree` only as a local pilot when the canonical file is expected to remain empty, enable it separately in every clone, re-certify the skill after changing the local behavior source, and back up the personalized content outside the worktree before disruptive Git operations. Avoid treating it as the final shared design. If user behavior sources become a supported feature, the best current direction is a tracked canonical behavior-source contract with an optional untracked user overlay whose existence, loading order, ownership, conflict policy, and audit treatment are represented explicitly by the blueprint system.
