## blueprint cleanup: mechanical default-wrapping can mis-nest section comments

**Context:** `skills/cloud-files/blueprint.yaml`, `skills/list-manager/blueprint.yaml` and similar blueprint files during shorthand-to-`default:` cleanup.

**Observation:** A mechanical rewrite that wraps top-level `patterns` / `allow_all_skills` / `allowed_callers` into `default:` can accidentally capture nearby 4-space section-divider comments as if they still belonged to the previous interface block.

**Impact:** The YAML can remain syntactically valid while the human-facing structure becomes misleading: interface-group boundaries appear to live inside the preceding interface, which makes later maintenance and review error-prone.

**Use:** After any bulk blueprint rewrite, visually inspect grouped interface comments and headings, not just schema/validator pass/fail. The safe fix is to move section-divider comment blocks back to the 2-space `script_interfaces` scope before considering the cleanup done.
